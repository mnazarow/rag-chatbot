# Модели: описание и рекомендации по выбору

Справочник по моделям, которые использует система: генеративная LLM (Ollama и vLLM),
эмбеддер и реранкер, базовая модель для дообучения. Те же рекомендации показываются
прямо в админ-панели под заголовками групп настроек «Генерация (LLM)» и «vLLM (GPU)»,
а также в блоке «Модель LLM».

Главное правило для RAG: **качество поиска (эмбеддер + чанкинг) важнее размера LLM.**
Хороший эмбеддер с моделью 8–14B обычно даёт лучшие ответы, чем гигантская LLM со
слабым поиском.

---

## 1. Генеративная LLM

Модель, которая формирует ответ из найденного контекста. Бэкенд выбирается полем
`LLM_BACKEND`:

- `ollama` — Apple Silicon (Metal) и CPU. Модели скачиваются по имени, например
  `qwen2.5:14b-instruct`. Имя активной модели — в поле `LLM_MODEL`.
- `openai` — vLLM на NVIDIA GPU (высокий throughput, OpenAI-совместимый API). Модель
  задаёт `VLLM_MODEL`, обращение идёт по `LLM_BASE_URL`.

### 1.1. Ollama — выбор по железу

| Память (ОЗУ/VRAM) | Рекомендуемые модели |
|-------------------|----------------------|
| 8–16 ГБ           | `qwen3:8b`, `qwen3.6:35b-a3b-q4_K_M` (MoE, быстрая), `gemma3:12b` |
| 24 ГБ             | `qwen3.6:35b-a3b` ✅ (баланс RU/скорость), `qwen2.5:32b`, `gemma3:27b` |
| 48–96 ГБ (Mac Studio) | `qwen2.5:72b`, `qwen3.6:27b`, `llama3.3:70b` |
| Слабый сервер/CPU | `qwen3:1.7b`–`qwen3:4b`, `llama3.2:3b` |

### 1.2. vLLM — выбор по VRAM

Для экономии видеопамяти берите квантованные сборки `-AWQ` / `-GPTQ-Int4`.

| VRAM | Рекомендуемые модели |
|------|----------------------|
| 24 ГБ (RTX 3090/4090) | `Qwen/Qwen2.5-14B-Instruct-AWQ` ✅, `Qwen/Qwen3-8B`, `Qwen/Qwen3.6-35B-A3B` (MoE Q4) |
| 48 ГБ (A6000 / 2×24)  | `Qwen/Qwen2.5-32B-Instruct-AWQ`, `Qwen/Qwen3-32B-AWQ`, `google/gemma-3-27b-it` |
| 80 ГБ (A100/H100)     | `Qwen/Qwen2.5-72B-Instruct-AWQ`, `Qwen/Qwen3-32B`, `hugging-quants/Meta-Llama-3.1-70B-Instruct-AWQ-INT4` |

Параметры контейнера vLLM:

- `VLLM_MAX_LEN` — макс. длина контекста (токенов). Больше = длиннее история/контекст,
  но выше расход VRAM. Для RAG разумно 8192–16384.
- `VLLM_TP` — tensor-parallel: число GPU, между которыми «распилена» модель. 1 для одной
  карты, 2+ для крупных моделей на нескольких GPU.

### 1.3. По задаче

- **Русский и RAG:** Qwen3.6 / Qwen3 / Qwen2.5 — самый сильный русский; Gemma 3 — 140+
  языков; Mistral-Small — точные фактологические ответы; Command-R — заточена под
  цитирование источников.
- **Reasoning** (Qwen3.6, DeepSeek-R1) — «думают» перед ответом: точнее на сложных
  вопросах, но медленнее. Для простого справочного RAG берите обычные instruct-модели
  или просите отвечать кратко.

Сменить модель: для Ollama — скачать/выбрать в блоке «Модель LLM»; для vLLM — выбрать в
выпадающем списке у поля `VLLM_MODEL` и нажать «Применить модель LLM (рестарт vLLM)».

## 2. Эмбеддер (`EMBED_MODEL`)

Кодирует документы и запрос в векторы для семантического поиска. Смена требует
переиндексации и согласованной `EMBED_DIM`.

| Модель | Размерность | Заметки |
|--------|-------------|---------|
| `BAAI/bge-m3` ✅ | 1024 | По умолчанию, многоязычный (dense+sparse), сильный RU |
| `Qwen/Qwen3-Embedding-0.6B` | 1024 | Топ MTEB 2026, лёгкий |
| `intfloat/multilingual-e5-large` | 1024 | Проверенный, хороший RU |
| `jinaai/jina-embeddings-v3` | 1024 | Длинные документы |
| `nomic-ai/nomic-embed-text-v2-moe` | 768 | Лёгкий многоязычный (⚠️ другая `EMBED_DIM`) |

⚠️ При смене эмбеддера поставьте корректную `EMBED_DIM`, затем «Сбросить индекс» и
«Переиндексировать».

**Реранкер** (`RERANK_MODEL`, по умолчанию `BAAI/bge-reranker-v2-m3`) — кросс-энкодер,
переоценивает топ-кандидатов и оставляет самые релевантные; главный рычаг качества.

## 3. Базовая модель для дообучения (`FINETUNE_BASE`)

QLoRA обучается на **не-квантованной fp16-базе**. Если поле пустое, база берётся из
`VLLM_MODEL` с отбрасыванием суффиксов `-AWQ`/`-GPTQ`/`-Int4`. Выбрать явно можно в
блоке «Модель генерации».

| VRAM | Базовые fp16-модели |
|------|---------------------|
| ~8–10 ГБ | `Qwen/Qwen2.5-1.5B-Instruct`, `Qwen/Qwen2.5-3B-Instruct`, `meta-llama/Llama-3.2-3B-Instruct` |
| ~16–18 ГБ | `Qwen/Qwen2.5-7B-Instruct` ✅, `Qwen/Qwen3-8B`, `google/gemma-2-9b-it`, `mistralai/Mistral-7B-Instruct-v0.3` |
| ~24 ГБ | `Qwen/Qwen2.5-14B-Instruct`, `Qwen/Qwen3-14B`, `google/gemma-3-12b-it`, `microsoft/Phi-4` |
| ~40 ГБ | `Qwen/Qwen2.5-32B-Instruct`, `mistralai/Mistral-Small-3.2-24B-Instruct-2506` |

Дообучение — дополнение к RAG (стиль/терминология), факты по-прежнему берутся из
документов. Подробнее — `finetune/README_finetune.md`.

## 4. Дефолтные модели по платформам

| Платформа | LLM по умолчанию | Бэкенд |
|-----------|------------------|--------|
| Apple Silicon (`setup.sh`) | `qwen3.6:35b-a3b-q4_K_M` (MoE, быстрая) | Ollama (Metal) |
| Linux + NVIDIA (`gpu_variant`) | `Qwen/Qwen3.6-35B-A3B` (MoE) | vLLM (CUDA) |
| Windows (нативно) | по умолч. `qwen3.6:35b-a3b-q4_K_M` (меняется `-Model`) | Ollama |
| Windows (Docker) | `qwen3.6:35b-a3b-q4_K_M` (MoE, 3B активных) | Ollama на хосте |

Везде по умолчанию — **Qwen 3.6** (MoE 35B с ~3B активных параметров: качество крупной
модели при скорости небольшой). Требует ~19–22 ГБ под веса (RAM для Ollama на CPU-хосте
или VRAM для vLLM). На машинах с малым объёмом памяти выберите модель поменьше в
админ-панели (напр. `qwen3:8b`, `gemma3:12b`).

Менять модель удобнее всего в админ-панели — значения в скриптах лишь стартовые.
