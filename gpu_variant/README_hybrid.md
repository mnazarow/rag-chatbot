# Расширенный режим GPU (hybrid+)

Дополнительный слой поверх основного GPU-варианта. Основа прежняя — **вектор +
реранк**, к ней добавляются три независимо включаемых улучшения. Всё управляется
из админ-панели (раздел «Расширенный поиск (hybrid+)»), отдельный сервер не нужен.

## Что добавляется

1. **Структурные метаданные (LLM).** При индексации каждый документ один раз
   прогоняется через LLM, который извлекает `product` (продукт/услуга),
   `topic` (тема) и `doc_type` (тип документа). Эти поля кладутся в Qdrant с
   индексами — вместе с уже имеющимися `category`, `date`, `ftype`. На GPU с
   vLLM это дёшево. Настройка: **LLM_METADATA** (scope: после переиндексации).

2. **Умные фильтры из вопроса (LLM).** Перед поиском вопрос разбирается LLM в
   структурированные условия (категория, продукт, период) и применяется как
   фильтр Qdrant. Если фильтр даёт мало результатов — автоматический откат к
   поиску без фильтра. Настройка: **SMART_FILTER** (на лету).

3. **Лёгкий граф-RAG.** Вопросы-«сводки» (сравнения, обзоры, связи между
   сущностями) маршрутизируются в граф знаний LightRAG; точечные вопросы
   по-прежнему идут через вектор+реранк. Граф строится на той же
   инфраструктуре — vLLM для извлечения и bge-m3 для эмбеддингов, без отдельного
   Ollama. Настройки: **GRAPH_RAG** (вкл/выкл) и **GRAPH_MODE** (mix/hybrid/…).

Архитектурно это и есть рекомендованная «золотая середина»: дешёвый и быстрый
вектор как основа, structured-метаданные для точности на прайсах/каталогах, и
граф — только там, где он реально нужен.

## Включение

Базовые улучшения 1–2 не требуют ничего, кроме включения в админке
(и переиндексации для метаданных). Для граф-RAG нужен LightRAG и построенный граф:

```bash
# на развёрнутом GPU-сервере
sudo bash gpu_variant/setup_hybrid.sh
```

Скрипт поставит LightRAG в существующее окружение и построит граф из `DOCS_DIR`.
Построение графа прогоняет документы через LLM (медленно) — для пробы укажите в
админке `DOCS_DIR` на подпапку из 30–50 файлов.

После этого в админ-панели включите нужные тумблеры:

- **LLM_METADATA** → затем кнопка «Переиндексировать»;
- **SMART_FILTER** → применяется сразу;
- **GRAPH_RAG** + **GRAPH_MODE** → применяется сразу.

## Стоимость и заметки

- LLM-метаданные добавляют ~1 вызов LLM на документ при индексации.
- Умные фильтры добавляют ~1 вызов LLM на каждый вопрос (небольшая задержка);
  если важна максимальная скорость — оставляйте выключенным и пользуйтесь
  быстрым rule-based авто-фильтром (AUTO_FILTER).
- Граф хранится в `graph_storage/` (не версионируется). Перестроить:
  `./.venv/bin/python -m graph_rag ingest`.
- Маршрутизация в граф — по ключевым словам вопроса (сравни, обзор, все,
  тенденция, связь и т.п.); список можно расширить в `graph_rag.py`.
