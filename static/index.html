Установка компонентов

      

        
        
        
      

      
Qdrant поднимается контейнером (нужен Docker). LightRAG ставится в окружение (pip). Статус и логи — в блоке «Состояние и операции».

    


    
    

      
Сайты в базе знаний

      
      

        
        
        
      

      
До 50 адресов. Страницы скачиваются, из них извлекается текст (в папку web/ внутри папки документов) и добавляется в базу с последующей индексацией.

Режим работы

      

      
Режим разом включает связку улучшений. Тонкая настройка флагов — в группе «Расширенный поиск (hybrid+)» ниже.

    


    
    

      
Движок ответов

      

      

        
        
      

      
«Векторный» — первый вариант (поиск + реранк). «LightRAG» направляет ВСЕ вопросы в граф знаний. Кнопка «Построить граф» ставит LightRAG и прогоняет документы через LLM (долго). При недоступности графа — автоматический откат на векторный.

    


    
    

      
Модель LLM

      

      

      
Каталог моделей (доступны к загрузке):

      

        
        
      

      

        
        
        
        
      

      
Сверху — установленные модели (выбор для генерации). Каталог — рекомендованные модели: для Ollama скачиваются, для vLLM имя подставляется в VLLM_MODEL (затем «Применить модель LLM»).

    


    
    

      
Модель генерации

      

      

        
        
        
      

      
Дообучение строит LoRA-адаптер из ваших документов (долго, грузит GPU). После применения переключите «Дообученная».

Удалённые хосты (Qdrant / LightRAG)

      

        

        

        

        

        

        

        

        

        

      

      

        
        
        
        
      

      

      

      
SSH по логину/паролю (paramiko). Linux: нужен root/passwordless sudo. macOS/Windows: Docker должен быть предустановлен, SSH включён. Пароли хранятся в remote_hosts.json — держите в доверенной сети. После «Переключиться»/«Вернуться» перезапустите сервис.

    


    
    

      
Калибровка по оценкам ответов

      

      

        
        
        
        
      

      

      
Анализирует оценки 👍/👎 и предлагает изменения (порог релевантности, объём выборки, температура, фильтры). «Откатить» возвращает значения до последней применённой калибровки.

    


    
    

      
Обновления

      

        
        
        
      

      

      
Сравнивает версию с GitHub и обновляет код + зависимости, затем перезапускает сервис. Работает, если установка делалась из GitHub (git-репозиторий).

    


    
    

      
Переустановка и сброс

      

        
        
        
        
        
        
      

      

        
        
        
        
      

      
«Переустановить окружение» — пересоздаёт .venv и зависимости, данные сохраняются. «Полная переустановка» — DESTRUCTIVE: удаляет окружение и ВСЕ данные (индекс, граф, адаптер, журнал) и разворачивает заново; `.env` сохраняется, сервис будет недоступен во время процесса. Выбирайте кнопку под вашу платформу. На GPU нужен сервис, запущенный от пользователя с правами sudo.
