# Корпоративный RAG-чатбот на локальной LLM

Чат-бот, отвечающий сотрудникам на вопросы по внутренним файлам (прайс-листы,
презентации, записи обучения и т.д.). Полностью локальный — данные не покидают
сервер. Работает на двух платформах с общим кодом: **Apple Silicon** (Mac Studio,
Ollama/Metal) и **Linux + NVIDIA GPU** (vLLM/CUDA). Управление целиком через
веб-панель: режимы (вектор / Hybrid+ / LightRAG), дообучение (LoRA), смена модели,
загрузка файлов, аналитика и переустановка.

## Деплой из GitHub на чистый сервер (GPU)

На чистом Linux-сервере с драйвером NVIDIA одной командой:

```bash
sudo -E REPO=https://github.com/USER/rag-chatbot.git ADMIN_TOKEN='пароль' bash deploy.sh
```

или, скачав скрипт прямо из репозитория:

```bash
curl -fsSL https://raw.githubusercontent.com/USER/rag-chatbot/main/deploy.sh \
  | sudo -E REPO=https://github.com/USER/rag-chatbot.git ADMIN_TOKEN='пароль' bash
```

`deploy.sh` клонирует репозиторий в `/opt/rag` и запускает полную GPU-установку
(Docker + NVIDIA toolkit, vLLM + Qdrant, Python-окружение, systemd-сервис
`rag-api`). После старта откройте `http://<ip>:8000` → раздел «Администратор» и
настройте папку с документами, модель и т.д. прямо в веб-панели.

Обновление до нового релиза (git pull + перезапуск):

```bash
sudo bash /opt/rag/update.sh            # с переиндексацией: sudo REINDEX=1 bash /opt/rag/update.sh
```

Для локального запуска на **Mac Studio** (Apple Silicon) деплой из GitHub не нужен —
см. раздел установки ниже (`setup.sh`).

## Как это устроено (архитектура)

Классический **RAG** (Retrieval-Augmented Generation): мы не дообучаем модель, а
на каждый вопрос находим релевантные куски документов и отдаём их LLM как контекст.
Это даёт точные ответы со ссылками на источники, мгновенное обновление при
изменении файлов и отсутствие «галлюцинаций» по ценам.

```
┌──────────────┐   индексация (ingest.py)        ┌───────────────┐
│  Папка с      │  парсинг → чанки → эмбеддинги    │   Qdrant       │
│  документами  │ ───────────────────────────────►│ (векторная БД) │
└──────────────┘  (PDF/DOCX/PPTX/XLSX/аудио)      └───────┬────────┘
                                                          │ поиск top-20
        вопрос                                            ▼
┌──────────────┐   /chat     ┌─────────────────┐   ┌───────────────┐
│  Веб-чат      │ ──────────► │  FastAPI (app)   │──►│ гибрид + реранк│
│ (браузер)     │ ◄────────── │  стриминг ответа │   │ (bge-reranker) │
└──────────────┘   ответ +   └────────┬─────────┘   └───────┬────────┘
                   источники           │ контекст + вопрос    │ top-6
                                       ▼                      │
                              ┌─────────────────┐ ◄───────────┘
                              │ Ollama + LLM     │
                              │ (Qwen2.5, Metal) │
                              └─────────────────┘
```

### Выбор компонентов и почему

**LLM — `Qwen2.5-32B-Instruct` (Q4) через Ollama.** На 96 ГБ это «золотая середина»
скорости и качества, отличный русский. При желании можно поднять `qwen2.5:72b`
(~42 ГБ, качественнее, медленнее) или специализированные русские модели
(`t-tech/T-pro-it`, Saiga) — меняется одна строка в `.env`. Ollama использует
Metal-ускорение Apple «из коробки», без возни с драйверами.

**Эмбеддинги — `BAAI/bge-m3`.** Многоязычная модель с сильным русским, размерность
1024, работает на MPS (Metal). Один из лучших открытых вариантов для смешанных
RU/EN корпусов.

**Реранкер — `BAAI/bge-reranker-v2-m3`.** Кросс-энкодер, который переоценивает
top-20 кандидатов и оставляет 6 действительно релевантных. Это главный рычаг
качества: именно реранк убирает «похожий, но не тот» контекст и даёт порог
`MIN_SCORE` для честного «не знаю».

**Векторная БД — Qdrant.** Быстрая, с фильтрами по метаданным и инкрементальным
обновлением. До тысяч файлов работает с запасом на одном сервере.

**Поиск — гибридный.** Плотный (семантический) поиск ловит смысл, BM25 —
точные совпадения (артикулы, названия товаров, числа в прайсах). Их комбинация
важна именно для прайс-листов.

### Особенности под ваши данные

- **Прайс-листы (XLSX/CSV)** разбираются построчно в формат `колонка: значение`,
  чтобы модель не путала цены между строками.
- **Презентации (PPTX)** индексируются по слайдам, включая заметки докладчика.
- **Записи обучения (аудио/видео)** автоматически транскрибируются Whisper
  (`mlx-whisper`, Metal) и индексируются как текст.
- **Сканы, фото и чертежи**: изображения (JPG/PNG/…) и RAW-фото (CR2/…) проходят
  OCR (Tesseract, rus+eng); чертежи DXF/DWG и 3D-CAD STEP/IGES — извлечение текста.
- **Письма, архивы и прочее**: Outlook `.msg`, старый `.doc`, HTML/MHTML, XML/JSON,
  SVG; архивы (ZIP/RAR/7Z/TAR) распаковываются, и содержимое индексируется.
- **Цитирование**: каждый ответ содержит источники (файл + страница/слайд).

## Установка на Apple Silicon (локально)

> Для Linux + NVIDIA и деплоя из GitHub см. [docs/INSTALL.md](docs/INSTALL.md)
> и раздел «Варианты деплоя» ниже.

```bash
# 1. Положить проект на сервер и запустить bootstrap
chmod +x setup.sh
./setup.sh            # Homebrew, Ollama, Qdrant, Python, скачивание моделей

# 2. Указать папку с документами
nano .env             # DOCS_DIR=/путь/к/документам

# 3. Проиндексировать
source .venv/bin/activate
python ingest.py

# 4. Запустить сервис
uvicorn app:app --host 0.0.0.0 --port 8000
```

Сотрудники открывают `http://<ip-сервера>:8000`.

## Эксплуатация

**Обновление индекса.** `python ingest.py` инкрементален — переиндексирует только
изменённые/новые файлы. Поставьте на расписание (см. `reindex.plist` — пример
launchd-задачи, запуск каждый час).

**Качество ответов** настраивается в `.env`: `TOP_K_RERANK` (объём контекста),
`MIN_SCORE` (строгость «не знаю»), `CHUNK_SIZE`. Промпт — в `prompts.py`.

**Доступ.** По умолчанию сервис слушает весь LAN. Для продакшена закройте за
reverse-proxy (Caddy/Nginx) с авторизацией или ограничьте firewall'ом.

## Веб-панель

После запуска на `http://<сервер>:8000` доступна панель (с темами оформления —
Тёмная / Cyberpunk 2077 / Лето):

- **Дашборд** — счётчики: всего/сегодня, доля ответов, релевантность, задержка,
  среднее время поиска и генерации, число чанков.
- **Схема работы и тесты** — актуальная схема связки + тестирование каждого
  компонента с общим вердиктом и логом.
- **Чат** — стриминг ответа + источники; оценка 👍/👎; галка «подробно об источниках
  и алгоритме»; приложить документ (📎) к вопросу. Каждая сессия — отдельный чат
  (свой `session_id`); кнопки «🆕 Новый чат» (очистить и начать заново) и
  «💾 Сохранить историю» (выгрузить переписку в Markdown-файл).
- **Журнал** — все запросы: вопрос, категория, релевантность, задержка, источники.
- **Аналитика** — по дням/категориям, топ источников, задержки, тайминги этапов,
  оценки ответов, частые слова.
- **Аналитика+** — графики по компонентам: Qdrant, граф, дообучение и тайминги
  (этапы, бенчмарк, длительности операций).
- **Система** — состояние и статистика Qdrant, LightRAG, дообучения, Hybrid+.
- **Загрузка сервера** — текущая загрузка хоста в реальном времени: CPU (общая и по
  ядрам), память и swap, диски, GPU (NVIDIA — загрузка/видеопамять/температура/
  питание; Apple — единая память), сеть, аптайм, топ процессов; автообновление 2 с.
- **Бенчмарк** — замер скорости компонентов (эмбеддер, реранк, поиск, Qdrant,
  LLM ток/с) с возможностью остановки.
- **Администратор** — всё управление (под `ADMIN_TOKEN`):
  - режим (Базовый / С метаданными / Hybrid+), движок (Векторный / LightRAG),
    модель генерации (базовая / дообученная);
  - выбор/загрузка модели LLM из каталога (Qwen3.6/Qwen3/Gemma3/Llama и др.) с
    рекомендациями и **прогрессом загрузки** (проценты, объём, скорость);
    настройки на лету (TOP_K, порог, температура, промпт);
  - выбор папки документов, загрузка файлов (Excel и др.), добавление сайтов
    (до 50, парсинг), переиндексация, построение графа, дообучение;
  - установка компонентов (Qdrant/LightRAG), калибровка по оценкам ответов;
  - очистка истории всех чатов и статистики (журнал запросов + оценки);
  - удалённые хосты по SSH (деплой, перенос данных, переключение);
  - проверка обновлений и обновление; сброс данных и переустановка.

## Варианты деплоя

- **Apple Silicon (Mac):** `setup.sh` локально, либо из GitHub —
  `mac_variant/deploy_mac.sh` (клон + установка + автозапуск через launchd).
- **Linux + NVIDIA (GPU):** `gpu_variant/run_gpu.sh` локально, либо из GitHub —
  `deploy.sh` (клон + vLLM/Qdrant + systemd). Генерация на vLLM (высокий throughput).
- **Windows:** `windows_variant/setup_windows.ps1` (PowerShell + winget + Ollama +
  Docker/Qdrant + Scheduled Task), либо `deploy_windows.ps1` из GitHub. Полностью
  контейнерный вариант — `windows_variant/docker/start_windows_docker.ps1`
  (приложение и Qdrant в Docker, генерация через Ollama на хосте).
- **Обновление:** `update.sh` (GPU) / `mac_variant/update_mac.sh` (Mac).
- **Переустановка:** окружения — `reinstall.sh` или кнопка в админке; полная —
  `reinstall_server.sh` / `mac_variant/reinstall_mac.sh`.
- **Разнесённое (split):** Qdrant + vLLM на отдельном GPU-сервере, приложение —
  на другом. `remote_variant/setup_backend.sh` + `remote_variant/setup_app.sh`
  (см. [remote_variant/README_remote.md](remote_variant/README_remote.md)).

Код приложения общий; платформа определяется настройками (`LLM_BACKEND`, `DEVICE`),
которые задаются скриптами и правятся в админке.
