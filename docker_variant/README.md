# Установка в Docker с Redis (Windows / Linux / macOS)

Единый Docker-вариант, в котором **Redis включён и настроен из коробки**. Поднимаются
три контейнера в одной сети:

- **qdrant** — векторная база (поиск);
- **redis** — кэш агрегатов статистики/аналитики (включён, лимит памяти + персистентность);
- **app** — само RAG-приложение (CPU).

Генерация ответов (LLM) выполняется через **Ollama, установленный на хосте** —
контейнер обращается к нему по адресу `host.docker.internal:11434` (на Linux это имя
пробрасывается автоматически через `extra_hosts`).

Папка проекта: `docker_variant/`. Все команды выполняйте из неё.

---

## Установка одной командой (автоматически)

Скрипты **сами устанавливают** Docker и Ollama (если их нет), скачивают модель,
создают конфиг/состояние и поднимают все контейнеры. Ничего ставить заранее не нужно.

### Windows
Дважды кликните **`start.cmd`** (или в терминале: `start.cmd`).
Установит Docker Desktop и Ollama через `winget`, скачает модель, соберёт и запустит
контейнеры (qdrant + redis + app), покажет чеклист и откроет браузер. Папка документов
по умолчанию — **`C:\db`** (создаётся автоматически).

```cmd
:: с паролем админки и своей папкой документов (необязательно):
start.cmd -DocsDir "C:\db" -AdminToken "ваш-пароль"
```
> Если это самая первая установка Docker Desktop, Windows может потребовать
> **перезагрузку** (WSL2/Hyper-V). После неё снова запустите `start.cmd` — он продолжит.

### Linux и macOS
```bash
chmod +x start.sh        # один раз
./start.sh
```
Linux: ставит Docker Engine (`get.docker.com`) и Ollama (`ollama.com/install.sh`) —
потребуется `sudo`. macOS: через Homebrew (`brew install --cask docker`, `brew install ollama`).

```bash
# своя папка документов / другая модель:
DOCS_DIR_HOST=/path/to/docs LLM_MODEL=qwen3:8b ./start.sh
# только запустить, ничего не устанавливая автоматически:
NO_AUTOINSTALL=1 ./start.sh
```

После старта откройте **http://localhost:8000**. Папка документов по умолчанию:
на **Windows** — `C:\db`; на **Linux/macOS** — `./docs` внутри `docker_variant/`
(создаётся автоматически; положите туда файлы и запустите переиндексацию в админке).

> Ручной запуск без скриптов (если Docker и Ollama уже есть):
> подготовьте `.env.docker` и `state/*`, затем `docker compose up -d --build`.

## Где проверить, что Redis работает

- Веб-интерфейс → раздел **«Система»** → панель **«⚡ Кэш Redis»**: версия, память,
  число ключей, клиентов, аптайм, процент попаданий (hit rate).
- Веб-интерфейс → **«Администратор»** → блок **«🗄 База данных и кэш»**: карточка Redis
  и кнопка «Очистить кэш».
- Из терминала: `docker compose exec redis redis-cli ping` → должно ответить `PONG`.

Кэш сбрасывается автоматически при каждом новом запросе, поэтому статистика/аналитика
не устаревают; Redis лишь ускоряет их выдачу.

## Настройка Redis

Параметры — в файле **`redis.conf`** (лимит памяти `maxmemory`, политика вытеснения,
персистентность AOF/RDB). По умолчанию **без пароля**: порт Redis наружу не публикуется,
он доступен только контейнерам в сети compose.

Включить пароль:
1. В `redis.conf` раскомментируйте `requirepass ВАШ_ПАРОЛЬ`.
2. В `docker-compose.yml` (сервис `app`) и/или в `.env.docker` задайте тот же
   `REDIS_PASSWORD=ВАШ_ПАРОЛЬ`.
3. Перезапустите: `docker compose up -d`.

## Полезные команды

```bash
docker compose ps                  # статус контейнеров
docker compose logs -f app         # логи приложения
docker compose logs -f redis       # логи Redis
docker compose down                # остановить (данные в томах сохраняются)
docker compose down -v             # остановить и УДАЛИТЬ данные (qdrant/redis/кеши)
```

Данные хранятся в именованных томах `qdrant_storage`, `redis_data`, `hf_cache`,
`previews_cache` и в папке `state/` (настройки админки, журнал SQLite, тайминги).
Резервные копии из админки складываются в `backups/`.

## Отличия от `windows_variant/docker`

Это расширенный вариант: добавлен контейнер **Redis** (включён и настроен) и единые
кросс-платформенные скрипты запуска для Windows, Linux и macOS. Образ приложения тот
же (`windows_variant/docker/Dockerfile`). Если Redis не нужен — используйте
`windows_variant/docker` (только Windows) или обычную установку из `docs/INSTALL.md`.

> Журнал и настройки по умолчанию в SQLite. При желании можно переключить на внешний
> MySQL/PostgreSQL в админке («База данных и кэш») — это не зависит от Docker-варианта.
