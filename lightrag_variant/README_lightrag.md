# Ветка LightRAG (граф-RAG) — для сравнения

Альтернативный ретривал на основе графа знаний. Цель — проверить на ваших
данных, даёт ли граф прирост на «сводно-аналитических» вопросах по сравнению
с основным вектор-вариантом. Это **отдельный стек**, не замена основного.

## Когда ждать выигрыша

LightRAG строит граф сущностей и связей и кластеризует их в «сообщества».
Это помогает на вопросах, требующих синтеза по многим документам:
«основные темы обучения за год», «как связаны продукт из презентации и
условия из договора», сводки и сравнения. На точечных фактах и прайсах
обычно паритет или проигрыш вектору + цена индексации заметно выше.

## Установка (отдельный venv)

```bash
cd lightrag_variant
python3.11 -m venv .venv-lr && source .venv-lr/bin/activate
pip install -r requirements-lightrag.txt

# модели в Ollama (LLM уже есть из основного setup; нужен ещё эмбеддер)
ollama pull bge-m3
```

В `.env` (можно общий с основным проектом) задайте `DOCS_DIR`, `LLM_MODEL`,
`OLLAMA_URL`. Дополнительно: `LIGHTRAG_DIR` (где хранить граф),
`OLLAMA_EMBED_MODEL=bge-m3`.

## Использование

```bash
python ingest_lightrag.py                       # построение графа (МЕДЛЕННО)
python query_lightrag.py "вопрос" --mode mix    # запрос
```

> Предупреждение по стоимости: индексация прогоняет каждый документ через
> LLM для извлечения сущностей. На ~1000 файлов локально это часы. Запускайте
> сначала на подвыборке (например, скопируйте 30–50 файлов в отдельную папку).

## Сравнение с вектор-вариантом

В корне проекта есть `compare.py` — прогоняет один список вопросов через оба
пайплайна и печатает ответы рядом, чтобы оценить разницу вручную.

## Версии

LightRAG активно развивается, API между версиями меняется. Зафиксирована
`lightrag-hku==1.3.0`. Если ставите другую версию и видите ошибку импорта
`initialize_pipeline_status` или сигнатуры `LightRAG(...)` — сверьтесь с
README соответствующего релиза и поправьте `rag_lightrag.py`.
